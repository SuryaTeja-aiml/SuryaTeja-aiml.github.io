Technical Projects

Concurrent Task Engine

C++17, POSIX Threads, Redis, Python
  • Built a multi-threaded C++ task execution engine using a fixed worker-pool architecture, avoiding per-task thread-creation overhead.
  • Designed a custom max-heap priority queue so higher-priority tasks automatically jump ahead of queued work.
  • Implemented thread-safe synchronization with std::mutex and std::condition_variable (predicate-checked to avoid spurious wakeups), releasing the queue lock before executing each task to prevent pool-wide bottlenecking.
  • Integrated the hiredis C client library directly into the engine for native Redis connectivity, avoiding any language-bridging overhead.
  • Synced live task metrics to a Python CLI dashboard through a Redis-backed, thread-safe metrics layer.

Distributed URL Shortener

Go, gRPC, PostgreSQL, Docker, Nginx
  • Built a fault-tolerant, multi-node URL shortener in Go that actively replicates every write to all healthy peer nodes via gRPC broadcast, keeping per-node PostgreSQL databases in sync.
  • Implemented heartbeat-based leader election — the lowest-ID online node is promoted to Leader, and killing the active leader's container triggers automatic failover, verified end-to-end.
  • Authored the Protobuf-defined gRPC contract for the inter-node replication service and generated the corresponding Go client/server stubs.
  • Designed an asynchronous click-analytics pipeline with buffered Go channels and background worker goroutines so redirects remain non-blocking.
  • Fronted the cluster with Nginx as a load balancer and a live dashboard showing node health, leader/follower roles, and link analytics; containerized full deployment with Docker Compose.

Real-Time Chat System

Java 21, Spring Boot 3, WebSocket/STOMP, PostgreSQL, Redis
  • Built a real-time chat backend with secure registration and login using BCrypt password hashing and JWT authentication.
  • Implemented bi-directional real-time messaging over WebSocket with STOMP/SockJS, secured by a custom JWT channel interceptor across both REST and WebSocket layers.
  • Designed REST endpoints for user search and paginated chat-history retrieval, indexed by chat session and timestamp for fast lookup.
  • Persisted chat history and user data in PostgreSQL while using Redis for session state and caching to keep message delivery low-latency.
  • Containerized the full stack (app, PostgreSQL, Redis) with Docker Compose for one-command local deployment.
